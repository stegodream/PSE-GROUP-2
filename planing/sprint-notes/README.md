## 🏃 Sprint
<details>
<summary><strong>Sprint Goal</strong></summary>

    > To complete the requirement analysis by identifying functional and non-functional requirements, defining system actors, assigning team roles, developing the Use Case Diagram, and creating initial wireframes based on the defined use cases.
</details>

<details>
<summary><strong>Sprint Duration</strong></summary>
 
    > Week 1 - Week 3
</details>

   <details>

<summary><strong>Tasks Selected</strong></summary>

    1. Conduct requirement analysis.
    2. Identify functional and non-functional requirements
    3. Define system actors.
    4. Assign team roles.
    5. Draw Use Case Diagram.
    6. Create wireframes according to use case diagram.
    7. Set up GitHub repository.
    8. Create Trello board.

</details>

<details>
<summary><strong>Work Completed</strong></summary>

    > During this sprint, the team successfully completed the requirement analysis and identified both functional and non-functional requirements.
    > System actors were clearly defined and represented in the Use Case Diagram.
    > We also assigned team roles based on individual strengths.
    > The GitHub repository and Trello board were set up to support project tracking and version control. 
    > Additionally, initial wireframes were created according to the interactions shown in the Use Case Diagram.
</details>

<details>
<summary><strong>Challenges Faced</strong></summary>
        
    > The team found it challenging to unify and finalise the Use Case Diagram due to different interpretations of system functionalities.
    > Communication during the wireframe design process was initially unclear.
    > Furthermore, most team members were using Figma for the first time, which required additional time to learn and slowed down the design process.
</details>

<details>
<summary><strong>Sprint Retrospective</strong></summary>

- **What went well:**
    
    > Team members were cooperative and actively participated in discussions.
    > Responsibilities were distributed clearly, and everyone contributed to completing the sprint tasks.
- **What didn't go well:**
    > Time estimation was inaccurate, and some tasks took longer than expected, especially learning new tools like Figma.
- **Improvement for Next Sprint:**

    > We will break larger tasks into smaller subtasks for better time estimation and manage our time more efficiently.
    > We will also improve communication during design discussions to avoid misunderstandings.
</details>
